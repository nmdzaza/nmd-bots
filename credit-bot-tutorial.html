This is the full guide for setting up the NMD Credit Bot (ScoreBoost) — a Telegram bot that handles your entire credit repair business automatically. Clients pay, upload their report, and get their dispute letters delivered as a PDF. You don't touch a thing.

💳 What the Credit Bot does

The Credit Bot is a fully automated credit repair service running inside Telegram. A client finds your bot, pays $30, uploads a photo of their credit report, and within minutes Claude AI analyzes the report and delivers a PDF full of custom dispute letters — all without you lifting a finger.

It handles:

  • Client intake — collects name, address, and credit report via Telegram
  • Payment — accepts Bitcoin directly or card-to-BTC via NOWPayments
  • AI analysis — Claude reads the credit report and identifies every negative item
  • Dispute letter generation — custom letters for Equifax, TransUnion, Experian (free), plus 15+ specialty bureaus
  • PDF delivery — sends a polished PDF packet directly to the client in Telegram
  • Add-on services — goodwill letters, debt validation, intent to sue, settlement analysis, card matching
  • Admin dashboard — review every client, approve/reject deliveries, export to CSV
  • Referral system — clients earn credit for referring others

The bot runs entirely on Python. It uses python-telegram-bot for the Telegram interface, SQLite for storing client data, Claude AI (claude-sonnet-4-6) for reading and analyzing the report, and fpdf2 for generating the PDF.

You (the admin) get a notification when a client pays and when a delivery goes out. You can review the generated letters before delivery or let it run fully automatically.

📋 What you need before you start
  • A computer with Python 3.10+ installed (Mac, Windows, or Linux)
  • A Telegram account
  • A free Anthropic API account (for Claude AI)
  • A Bitcoin wallet address to receive payments
  • About 20 minutes
You don't need to know how to code. Every command in this guide is copy-paste. If you can open a terminal and paste text, you can run this bot.
1
Create your Telegram bot with BotFather

Every Telegram bot starts with @BotFather — Telegram's official bot creation tool. This gives you the bot token your code needs to connect to Telegram.

Steps:

  • Open Telegram and search for @BotFather
  • Send the message /newbot
  • Choose a display name — something like ScoreBoost Credit Repair
  • Choose a username — must end in "bot", e.g. ScoreBoostByNMDBot
  • BotFather gives you a token — it looks like this:
7123456789:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Save this token. You'll need it in Step 7. Never share it publicly — it gives full control of your bot.

While you're in BotFather, also run these two commands to polish the bot:

# Set a description (shows on the bot's profile)
/setdescription → paste: "AI-powered credit repair. Upload your report, get dispute letters delivered instantly."

# Set profile photo (optional but professional)
/setuserpic → send a logo image
2
Get your Telegram User ID

The bot needs your Telegram user ID so it knows who the admin is. Your user ID is a number like 5258417842 — not your username.

  • Open Telegram and search for @userinfobot
  • Send it any message
  • It replies with your user ID — copy that number
Your admin ID controls who can access the /admin panel. Only your ID gets in — no one else can see client data or approve deliveries.
3
Get your Anthropic API key

The bot uses Claude Sonnet (claude-sonnet-4-6) to read credit reports and write dispute letters. You need an Anthropic API key.

  • Go to console.anthropic.com and create a free account
  • Click API Keys in the left sidebar
  • Click Create Key — name it "credit-bot"
  • Copy the key — it starts with sk-ant-...
Anthropic gives new accounts $5 in free credits. One credit report analysis costs roughly $0.03–0.08 depending on length. That's 60–160 free analyses before you pay anything.
sk-ant-api03-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
4
Set up a Bitcoin wallet for payments

Clients pay $30 in Bitcoin. The bot verifies payments automatically using the Blockcypher API — no manual screenshot checking needed.

  • Download Cash App, Coinbase, or any Bitcoin wallet
  • Create your account and find your Bitcoin receive address — it looks like bc1q... or 1A1zP...
  • Copy that address — you'll paste it into your .env file
Want to accept credit cards too? Sign up at nowpayments.io (free) and grab your API key. This lets clients pay by card and you still receive BTC. Add it as NOWPAYMENTS_API_KEY in your .env.
5
Download the bot files

The bot runs from two Python files: credit_bot.py (the Telegram interface) and claude_credit.py (the AI engine). Get them from the NMD GitHub.

Terminal
# Clone the repo
git clone https://github.com/nmdzaza/nmd-bots.git

# Or download just the credit bot folder
cd ~/Desktop
mkdir credit-bot && cd credit-bot

Your folder should contain:

credit-bot/
├── credit_bot.py        # main bot — runs the Telegram interface
├── claude_credit.py     # AI engine — Claude analysis + PDF generation
├── requirements.txt     # Python dependencies
└── .env                 # your secrets (you create this in Step 7)
6
Install Python dependencies

The bot needs a few Python packages. Install them all at once:

Terminal — run from inside the credit-bot folder
# Make sure Python 3 is installed
python3 --version

# Install all dependencies
pip3 install python-telegram-bot python-dotenv httpx anthropic fpdf2

# Or if there's a requirements.txt
pip3 install -r requirements.txt
If you're on a Mac and get permission errors, add --break-system-packages to the end of the pip command. If you're on Windows, use pip instead of pip3.
7
Create your .env file

The .env file holds all your secret keys and config. Create it in the same folder as credit_bot.py. Never put this file on GitHub.

credit-bot/.env — minimum required to run
# ── Required — bot won't start without these ──────────────────
BOT_TOKEN="7123456789:AAFxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
ADMIN_ID="5258417842"              # your Telegram user ID from Step 2
BOT_USERNAME="ScoreBoostByNMDBot"   # your bot's @username (no @)
BTC_ADDRESS="bc1qxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
ANTHROPIC_API_KEY="sk-ant-api03-xxxxxxxxxx"

# ── Optional — adds card payment support ──────────────────────
NOWPAYMENTS_API_KEY=""           # get at nowpayments.io (free account)

# ── Optional — add a Telegram channel for delivery receipts ───
CHANNEL_ID=""                     # your channel's chat ID (leave blank to skip)

How to get your channel ID (optional):

  • Add @userinfobot to your Telegram channel as an admin
  • Send any message in the channel — the bot replies with the channel's ID
  • It looks like -1001234567890 (starts with -100)
Never commit your .env file to GitHub. Add .env to your .gitignore file if you're using Git.
8
Run the bot

You're ready. Open your terminal, navigate to the credit-bot folder, and run:

Terminal
cd ~/Desktop/credit-bot
python3 credit_bot.py

You should see output like this:

2026-03-19 12:00:01 INFO Database initialized.
2026-03-19 12:00:01 INFO Config validated. BOT_TOKEN ✓ ADMIN_ID ✓ BTC_ADDRESS ✓
2026-03-19 12:00:02 INFO Application started. Bot is running as @ScoreBoostByNMDBot
The bot creates a SQLite database file (credit-bot.db) automatically on first run. All client data is stored there.

If you see errors, jump to the Troubleshooting section at the bottom.

10
Keep it running 24/7 with PM2

Right now the bot stops when you close your terminal. To keep it running forever — even after restarts and crashes — use PM2.

Terminal — install PM2 and start the bot
# Install PM2 (one time)
npm install -g pm2

# Start the credit bot with PM2
pm2 start credit_bot.py --interpreter python3 --name credit-bot

# Make it restart on server reboot
pm2 startup
pm2 save

Useful PM2 commands:

pm2 list                  # see all running bots
pm2 logs credit-bot       # watch live logs
pm2 restart credit-bot    # restart after code changes
pm2 stop credit-bot       # stop the bot
Running on a Mac? PM2 works perfectly. If you want it running even when the laptop is closed, consider a cheap VPS — DigitalOcean, Hetzner, or Vultr start at $4–6/month. Your bot runs 24/7 in the cloud.

👤 What your client sees

Here's the exact experience from your client's perspective, step by step:

  • /start — Bot sends a welcome message explaining the service and pricing
  • Intake form — Client enters their full name and mailing address (used on dispute letters)
  • Report upload — Client takes a photo of their credit report and sends it to the bot
  • Service selection — Client picks which bureaus and add-ons they want
  • Payment — Bot sends a BTC address with the exact amount to send. Client pays, blockchain confirms automatically within 5–10 minutes
  • Processing — Bot sends "Working on your letters…" — Claude is analyzing the report
  • Delivery — Bot sends the PDF dispute packet directly in Telegram. Done.
The entire flow from payment to delivery takes 3–8 minutes depending on report length. No human involvement required.
🔧 Troubleshooting

Bot won't start — "BOT_TOKEN not set"

Your .env file isn't being read. Make sure it's in the same folder as credit_bot.py and that the file is named exactly .env (with the dot, no extension).

# Check your .env is there
ls -la | grep .env

Claude API error — "invalid_api_key"

Double-check your ANTHROPIC_API_KEY in your .env file. Make sure there are no spaces or extra quotes around the key.

Payment not being detected

BTC payment detection uses the Blockcypher API and looks back 2 hours for transactions. Make sure:

  • The BTC address in your .env exactly matches where the client sent money
  • The transaction has at least 1 blockchain confirmation (usually 10–20 min)
  • You're not on a VPN that blocks blockchain API calls

PDF is blank or missing letters

Claude couldn't read the credit report image. Make sure the client uploads a clear, well-lit photo — not a scan of a scan. The bot accepts JPG, PNG, and HEIC formats.

Bot crashes after running a while

Use PM2 (Step 10) — it auto-restarts the bot on any crash. Check pm2 logs credit-bot to see the error that caused the crash.
